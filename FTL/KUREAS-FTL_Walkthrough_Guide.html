Exercise 1: Importing a SAPHIRE Fault Tree Model

In this exercise you will load an existing SAPHIRE fault tree model with all its companion files and explore the resulting visualization.

1

Open KUREAS-FTL.html in your browser. You will see the “Open Fault Tree and Related File(s)” upload screen.

2

Multi-select your SAPHIRE files. Click the file input and select all companion files at once: the .FTL file (required), plus .BED (descriptions), .BEI (probabilities), .GTD (gate types), and .FTC (cut sets). Hold Ctrl (or Cmd on Mac) to select multiple files.

3

Explore the canvas. The fault tree renders on an interactive canvas. Use the mouse wheel to zoom in/out, click and drag to pan, and click any node to select it. The status bar at the bottom shows navigation hints.

4

Switch between trees. If the model contains multiple fault trees, use the tree selector dropdown in the toolbar to switch between them. Each tree has its own top event and logic structure.

5

Expand Transfer gates. Transfer gates (shown with a triangle symbol) reference other fault trees. Click on a Transfer gate to see where it links. KUREAS-FTL automatically resolves these references to display the complete logic.

6

Save as a KUREAS project. Click Save FTL to save the imported model as a .KNOW_FTL file. This native format preserves all data and allows faster loading next time.

About .KNOW_FTL files Unlike the SAPHIRE MAR-D format which splits data across multiple files, the .KNOW_FTL format stores everything — tree logic, descriptions, probabilities, notes, and change history — in a single JSON file.

Exercise 2: Building a Fault Tree from Scratch

This exercise walks you through creating a new fault tree project and building logic manually.

1

Start a new project. On the upload screen, click the purple Start New Project button. Enter a project name when prompted (e.g., PUMP SYSTEM). You will see an empty canvas.

2

Add the first fault tree. Click New FT in the toolbar. Enter the top event name (e.g., PUMP-SYS-FAIL), select a gate type (OR), and optionally add a description. The top gate appears on the canvas.

3

Add child gates. Click the top gate to select it, then click New Gate. Enter the gate name (e.g., TRAIN-A-FAIL), select AND as the type, and add a description. Repeat to add TRAIN-B-FAIL as another AND gate under the top OR gate.

4

Add basic events. Select TRAIN-A-FAIL and click New BE. Enter the event name (e.g., PUMP-A-FTS for “Pump A Fails to Start”), a description, and a probability (e.g., 3E-3). Add another basic event VALVE-A-FTO (Valve A Fails to Open, probability 1E-3).

5

Complete Train B. Select TRAIN-B-FAIL and add basic events PUMP-B-FTS (3E-3) and VALVE-B-FTO (1E-3). Your tree now represents a two-train redundant pump system where both trains must fail (OR of two AND gates).

6

Review probabilities. Open Settings and set Show Probability to “2 Digits (1.23E-5)” to display calculated probabilities on every node in the tree.

Checkpoint Save your project as a .KNOW_FTL file. Your two-train pump system tree should show the correct Boolean logic with probabilities propagated through the gates.

Exercise 3: Editing Nodes and Using Scope-Aware Changes

This exercise demonstrates editing existing nodes, handling shared events, and using undo.

1

Edit a basic event. Click on PUMP-A-FTS to open the edit panel. Change the probability to 5E-3 and add a note: “Updated per latest plant data.” Click Apply. Notice the probability updates on the canvas.

2

Undo the change. Press Ctrl+Z (or click the Undo button). The probability reverts to 3E-3. Press Ctrl+Z again to verify multi-level undo works. Redo by making the change again.

3

Create a shared basic event. Add a new basic event under TRAIN-A-FAIL called POWER-LOSS (probability 1E-4). Then add another basic event under TRAIN-B-FAIL with the same name POWER-LOSS. KUREAS-FTL recognizes this as a shared event — changing it in one location affects both.

4

Test scope-aware editing. Click the POWER-LOSS event under Train A and change its probability to 2E-4. A scope dialog appears asking: apply to this instance only, or to all instances? Select All instances. Verify that the probability updates in both locations.

5

View the change log. Click View Changes in the toolbar. The change log shows every edit you have made, with timestamps, old values, and new values. This provides a complete audit trail for QA review.

Name conflicts KUREAS-FTL prevents you from giving a basic event and a gate the same name. If you try to rename a basic event to a name already used by a gate, you will see a conflict warning.

Exercise 4: Using Floating Windows and Cut Sets

This exercise covers the Basic Event List and Cut Sets floating windows.

1

Open the Basic Event List. Look for the BE List button in the toolbar. A draggable, resizable floating window appears listing all basic events in the project with their descriptions and probabilities.

2

Edit from the list. Click any event in the BE List to expand its editor. Change the description or probability directly in the floating window. Changes propagate to the canvas immediately.

3

Drag-and-drop. From the BE List, drag an event name and drop it onto a gate on the canvas. This adds the basic event as a child of that gate — useful for quickly connecting existing events to new logic.

4

View cut sets. If cut set data was loaded from a .FTC file, the Cut Sets floating window shows each minimal cut set with its probability contribution, sorted from highest to lowest. The window header shows the total fault tree probability.

Floating window tips Both windows can be dragged by their title bar, resized from the edges, and minimized. They stay on top of the canvas so you can reference them while editing.

Exercise 5: Exporting and Reporting

This exercise covers all output options.

1

Export to SAPHIRE. Click Export. Select format = SAPHIRE MAR-D (.FTL). Check the trees you want to include. Click Export. The downloaded .FTL file can be loaded directly into SAPHIRE.

2

Export to OpenPSA. Click Export again and select OpenPSA MEF (.xml). This generates an XML file conforming to the Open Probabilistic Safety Assessment Model Exchange Format, which can be imported into other PSA tools that support the standard.

3

Export selected trees. Click Export and select KUREAS-FTL (.KNOW_FTL). Uncheck some trees to export only a subset. The resulting .KNOW_FTL file contains only the selected trees and their referenced basic events.

4

Generate a report. Click Report. Select which trees to include, the output format (Word .doc or Markdown), and whether to include cut sets. Set a cut set display limit if the model is large. Click Generate. The report includes tree diagrams rendered as images, event listings, and probability calculations.

5

Customize the view for screenshots. Before generating reports, open Settings and configure the visual style: try different backgrounds (engineering green paper is popular for print), color modes, and line styles. Reports capture the tree exactly as it appears on the canvas.

Report preferences are saved Your report format, tree selection, and header options are saved in browser localStorage and restored next time you open the Report dialog.
