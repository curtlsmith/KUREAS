KUREAS-REPORT

Knowledgebase with Uncertainty for Risk Engineering Analysis of Systems — Report Generation Module
v1.0
MIT License · Single-file, browser-based
📋 Overview
KUREAS-REPORT assembles multi-source PRA/reliability reports by dragging data elements from loaded KUREAS files into a Report Creation Area, then exporting to Word (.docx) or Markdown (.md).
Layout
◀Left pane — Report Elements library (Basic Elements, Insertable Elements, loaded files)
▶Right pane — Report Creation Area (ordered list of elements to include)
⟺Drag the divider to resize panes
Tip: Drop .KNOW_RPT, .KNOW_HA, .KNOW_FTL, or .ELEMENTS files directly onto the left pane to load them.
🔧 Toolbar Buttons
Open REPORTLoad a saved .KNOW_REPORT project file
Save REPORTSave current report to .KNOW_REPORT (dialog prompts for name; no "REPORT_" prefix added)
Open KUREASLoad one or more source files (.KNOW_RPT / .KNOW_HA / .KNOW_FTL / .KNOW_3D)
ReferencesEdit the citation library; cited refs auto-populate the References section
GenerateOpen the Report Preview window with Save as Word / Save as Markdown buttons
⚙App settings — font, page size, preview size, numeric precision

✨ Basic Elements
Click any tile to add to the Report Creation Area.
⚙️Formatting — page layout, margins, header/footer, page numbers, numbered section styles
📄Cover Page — title, subtitle, author, org, date (text), custom content
🔖Section Header — H1–H6 heading with optional body text
📋Table of Contents — Word auto-TOC (H1–H4); updates on open
🖼List of Figures — auto Figure list from SEQ fields
📊List of Tables — auto Table list from SEQ fields
🔤Acronyms — scans report text for Expansion (ABBR) patterns; click ↻ to refresh
📚References — cited references (only entries cited in the text are included)
📎File Insertion — placeholder for an external file path in the final document
Formatting stays locked at position #1. Acronyms re-scans all report text each time ↻ is clicked.
📦 .KNOW_RPT Sections
When a .KNOW_RPT is loaded, its available sections appear in the left pane under the filename.
Drag Options
📦"All Information in…" card (indigo, top of file) — drops a single collapsed RPT group containing all sections
📋Individual section — drag any section tile to add just that piece
RPT Group in Report Area
▶Click the chevron to expand and see all contained sections
🔓Ungroup button — expands group into individual section rows in place
↕Move, view, or remove the group as a single unit
Updating After Reload
If a .KNOW_RPT file is reloaded and matching elements already exist in the report area, a dialog asks whether to Update Elements (refreshes data in place, preserving position) or Keep Existing.
🗂 Report Creation Area
Adding Elements
🖱Drag elements from the left pane and drop onto the list or the "Drop elements here" zone at the bottom
➕Click Basic Element tiles to add directly
Reordering
↕Drag any element card to a new position (blue indicator shows drop slot)
▲▼Use Up/Down arrow buttons as an alternative
Element Actions
👁Eye — full preview of element content
✏Pencil / ↻ — edit (Cover Page, Section Header, Formatting, File Insertion) or refresh (Acronyms)
🗑Trash — remove from report
New elements are appended at the bottom; the area auto-scrolls to show them.
⚙️ Document Formatting Element
Page LayoutPortrait or Landscape
Page NumbersNone · Top/Bottom × Left/Center/Right
MarginsTop / Bottom / Left / Right (inches)
HeaderText on every page header
FooterText on every page footer
Numbered StylesAuto-numbers headings: 1. / 1.1. / 1.1.1. etc.
Formatting is always placed at position #1 and applies to the entire .docx output. It is invisible in the HTML preview.
📄 Word (.docx) Export Features
📋Table captions appear above each table — "Table N. Description." using Word SEQ fields
🖼Figure captions appear below each image — "Figure N. Description." using Word SEQ fields
📐Image sizing — actual pixel dimensions read from PNG/JPEG headers; auto-scaled to fit page width
📑TOC / TOF / TOT use Word's native field mechanism; update with Ctrl+A → F9 in Word
🔤Acronym table rendered as a two-column, alphabetically sorted table
📚References — only entries whose citation key appears in the document text are included
Caption numbers are SEQ fields — Word renumbers them automatically when fields are updated.
📝 Insertable Elements
Custom rich-text blocks created or imported externally.
📥Load button — import a .ELEMENTS file (duplicate detection with merge dialog)
✏Edit button — open the Insertable Elements editor to create, edit, or export
💾Save & Export — saves the library as a .ELEMENTS file for sharing
🖱Drag any insertable element to the Report Creation Area or drop a .ELEMENTS file onto the left pane
⚡ Typical Workflow
1. Open KUREAS file(s) → 2. Set Formatting → 3. Add Cover Page
4. Drag sections → 5. Add TOC / TOF / TOT → 6. Add Acronyms
7. Add References → 8. Generate → Save .docx
Save REPORT at any time to preserve your layout as a .KNOW_REPORT for later editing.
💡 Tips & Notes
🔄Reload a .KNOW_RPT after re-exporting from KUREAS-SYS — a dialog offers to update existing report elements in place
🔤Click Acronyms ↻ after the report is fully assembled to capture all acronyms across all sections
👁Use Settings → Show Preview to control inline previews (None / Small / Large) — improves performance on large reports
📐Set font, page size, and decimal precision in Settings; these apply to all output formats
📑In Word, press Ctrl+A then F9 after opening the .docx to update all TOC, TOF, TOT, and SEQ fields
🖱The Ungroup button on an RPT group lets you reorder or remove individual sections after dropping the whole file
